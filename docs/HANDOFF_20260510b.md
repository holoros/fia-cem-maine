## gr_ratio Layer 2 status (see GR_RATIO_LAYER2_AUDIT.md)

Root cause located at `R/03_harvest_choice.R` line 409: `vol_removed_total = volcfnet * tpa_live * harvest_intensity` double counts the per acre conversion already applied at `R/01_data_prep.R` line 149 (`volcfnet = sum(TPA_UNADJ * VOLCFNET, na.rm = TRUE)`). Patch documented in the audit doc.

Coverage caveat: the function is on the economic overlay path (`--use_maine_econ`), not the current multistate flag set. So this fix is required for the next ME r21 econ rerun and any future state economic scenarios, but does not affect the in flight six runs. The currently running outputs will carry post Layer 1 gr_ratio values (around 0.005 to 0.006 for MN style harvest rates).
