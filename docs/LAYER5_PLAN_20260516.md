# Layer 5 plan: replace the dVAL revenue proxy with proper Wear 2025 differential

*Generated 16 May 2026 after the Layer 4 patch was applied. Captures the forward plan for the next session, conditional on the Layer 4 smoke result.*

## The remaining issue

`R/03_harvest_choice.R` line 257 reads:

```r
# Compute dVAL (simplified: using revenue as proxy)
dVAL = REV_harvest,
```

This is a placeholder shortcut. The Wear and Coulston (2025) paper defines:

> dVAL = |REV + (1+d)^-n * (EV_h - EV_nh)|

Where:
- `REV`: revenue from harvest now
- `EV_h`: expected ending inventory value if harvested (regen value after 5 years)
- `EV_nh`: expected ending inventory value if not harvested (standing timber after 5 years)
- `delta = (1+d)^-n`: discount factor for the remeasurement period n (typically 5 years at rate d)

Using REV directly inflates dVAL because:
- REV alone reflects total stand value
- The proper dVAL is the *differential* between harvesting now vs waiting, which is much smaller because EV_h and EV_nh are large opposite valued terms that mostly cancel

For a mature stand worth $2,000/ac:
- REV = $2,000
- EV_h ≈ $50 (residual regen)
- EV_nh ≈ $2,200 (5 years growth at 1-3 percent)
- delta = 0.822 (4 percent annual discount, 5 yr)
- dVAL = abs(2000 + 0.822 * (50 - 2200)) = abs(2000 - 1767) = $233

So the proper dVAL is roughly 1/10 of REV for a typical Maine stand. Using REV as proxy inflates the logit term by 10x.

Combining the Layer 4 fix (200x price reduction) with Layer 5 (10x dVAL proxy correction), total dVAL would drop by 2000x from the pre-Layer-2/3/4 baseline. Logit term: 0.0017 × $233 = 0.40. With intercept -1.78: xb = -1.38, P(harvest) = 0.20.

That's still above observed Maine 10 percent, but within a factor of 2 — which suggests the residual gap could be explained by:
- Variation in stumpage price (Maine prices may be lower than the $250/MBF / $12/cord defaults)
- The Wear 2025 coefficients being calibrated against a slightly different stand population
- Owner stratification (`--use_owner_balanced`) further suppressing harvest
- The proper differential including additional cancellation effects (county harvest offset, climate response)

## Layer 5 patch design

Two components needed:

### 1. Make compute_ending_value() actually work

Currently the function `compute_ending_value()` (R/03 line 99) is dead code. It computes a single EV value without distinguishing harvest vs no-harvest. To support Wear 2025 dVAL properly, we need:

```r
compute_ending_value <- function(cond_data, prices, discount_rate = 0.04,
                                  remper = 5) {
  MBF_per_CUFT  <- 1 / 200
  CORD_per_CUFT <- 1 / 80

  cond_data |>
    mutate(
      # EV if harvested: residual stand value (mostly $0, regen with no
      # merchantable timber at year 5). Set to a small constant.
      EV_h = 50,    # $/ac residual value for regen stand

      # EV if not harvested: standing timber value at year T2 (cuft/ac to MBF/cord
      # to $/ac). The T2 columns are at +5 yr from baseline, which IS the remper.
      EV_nh_sawtimber = coalesce(T2_vol_sawtimber_softwood, 0) * MBF_per_CUFT *
                          prices$sawtimber$softwood +
                        coalesce(T2_vol_sawtimber_hardwood, 0) * MBF_per_CUFT *
                          prices$sawtimber$hardwood,
      EV_nh_pulpwood  = coalesce(T2_vol_pulpwood_softwood, 0) * CORD_per_CUFT *
                          prices$pulpwood$softwood +
                        coalesce(T2_vol_pulpwood_hardwood, 0) * CORD_per_CUFT *
                          prices$pulpwood$hardwood,
      EV_nh = EV_nh_sawtimber + EV_nh_pulpwood
    )
}
```

### 2. Call it from predict_harvest_probability

Replace line 257 in `predict_harvest_probability`:

```r
# Old:
dVAL = REV_harvest,

# New: proper Wear 2025 differential
.ev_data = compute_ending_value(cond_data, prices),
delta = (1 + 0.04)^(-5),
dVAL = abs(REV_harvest + delta * (.ev_data$EV_h - .ev_data$EV_nh)),
```

The `.ev_data` join needs care because `compute_ending_value()` returns a tibble; either compute its outputs inline or merge by PLT_CN/CONDID.

A simpler implementation: compute EV_nh inline in the mutate block of predict_harvest_probability, skip the function call:

```r
# Inline Layer 5 in predict_harvest_probability:
mutate(
  ...
  EV_nh = (coalesce(T2_vol_sawtimber_softwood, 0) * 0.005 * prices$sawtimber$softwood +
           coalesce(T2_vol_sawtimber_hardwood, 0) * 0.005 * prices$sawtimber$hardwood +
           coalesce(T2_vol_pulpwood_softwood,  0) * 0.0125 * prices$pulpwood$softwood +
           coalesce(T2_vol_pulpwood_hardwood,  0) * 0.0125 * prices$pulpwood$hardwood),
  EV_h  = 50,
  delta = (1 + 0.04)^(-5),
  dVAL = abs(REV_harvest + delta * (EV_h - EV_nh)),
  ...
)
```

(0.005 = 1/200 MBF/cuft, 0.0125 = 1/80 cord/cuft.)
