# CEM pipeline patch: v4 productivity multiplier integration

These four files comprise the patch that wires the Phase 2 v4 productivity multiplier into the live CEM pipeline at `~/fia_cem_projections/` on OSC Cardinal. The integration is gated behind a new CLI flag `--use_v4_prod_mult` so default behavior (r20 and earlier) is unchanged.

## Files

`06_projection_engine.R` adds the v4 multiplier loader and applier (about 60 lines, lines 686 to 745 in the patched file). The multiplier is applied **before** the SDImax cap so over-scaled strata still get bounded by carrying capacity.

`run_projection.R` adds two CLI flags: `--use_v4_prod_mult` (boolean, default off) and `--v4_prod_mult_strength` (numeric, default 1.0).

`submit_r21_smoke.sh` is a 1 sim, 5 cycle smoke test that verified the integration on Cardinal as job 9182112 (completed in 2:06, no errors, multiplier loaded for all 5,027 plots with per-cycle range [0.985, 1.025] and mean 0.999).

`submit_rcp45_wear_r21.sh` is the full r21 submit script for the rcp45 wear scenario (BAU baseline). Submitted as job 9182685; expected runtime 3 to 4 hours actual within the 20 hour walltime envelope.

## Behavioral semantics

The multiplier is applied per cycle as `prod_mult^(strength / 30)`, where 30 is the design horizon for the cumulative effect. With `strength = 1.0`, a stratum with `prod_mult = 1.5` gets `1.5^(1/30) = 1.0136` per cycle, accumulating to exactly 1.5x over a 30 cycle projection. The lookup is keyed by `PLT_CN` via `config/yc_plot_membership.csv` (5,027 forest plot rows). Plots not present in the membership table default to `prod_mult = 1.0`.

The treatment dimension is collapsed to `untreated` for the projection: the CEM pipeline applies harvest dynamically through scenario logic, so the static treatment classification from FIA history would double count harvest effects. Future v5 work could expose treatment as a runtime classifier.

## Usage

To enable the multiplier on a scenario:

```bash
Rscript run_projection.R [usual flags] \
  --use_v4_prod_mult --v4_prod_mult_strength 1.0
```

The strength parameter scales the deviation from neutrality. `strength = 0.5` halves the multiplier effect (cumulative 1.5^0.5 = 1.22 over 30 cycles); `strength = 0` is equivalent to omitting the flag.

## r21 SLURM scripts

Four r21 submit scripts were generated by sed-substituting `r20` for `r21` in the existing r20 scripts and inserting the `--use_v4_prod_mult --v4_prod_mult_strength 1.0` flags. They live in `~/fia_cem_projections/osc/`:

- `submit_rcp45_wear_r21.sh` (submitted, job 9182685)
- `submit_rcp45_wear_econ_r21.sh` (staged)
- `submit_rcp85_wear_r21.sh` (staged)
- `submit_rcp85_wear_econ_r21.sh` (staged)

Launch the rest with `sbatch osc/submit_<scenario>_r21.sh` from `~/fia_cem_projections/`. After all four complete, run `osc/expand_r21.sh` (analogous to `expand_r20.sh` after sed substitution) to produce statewide AGC trajectories. A side by side r20 vs r21 comparison figure reveals whether the empirical asymptote signal is captured by the existing pipeline (similar trajectories) or represents missing site productivity differentiation (divergent trajectories).

## Diagnostic plan

The r20 vs r21 comparison answers a manuscript-relevant question: does the CEM pipeline's existing donor-driven growth machinery already capture the empirical productivity gradient that v4 represents, or is v4 adding new information?

If r21 statewide AGC differs from r20 by less than about 5 percent, the existing pipeline already captures the signal and the v4 multiplier is redundant for state-level projections (though still useful for stratum level inference). If r21 differs by 10 percent or more, the multiplier is adding meaningful site productivity signal that the donor pool was missing, which strengthens the manuscript argument for stratum aware projection methods.

## Rollback

If the patch causes problems, restore from the backup:

```bash
cp ~/fia_cem_projections/R/06_projection_engine.R.bak.20260505_161709 \
   ~/fia_cem_projections/R/06_projection_engine.R
cp ~/fia_cem_projections/run_projection.R.bak.20260505_161710 \
   ~/fia_cem_projections/run_projection.R
```

The v4 multiplier CSV in config (`config/cem_productivity_multipliers_v4.csv`) and the membership file (`config/yc_plot_membership.csv`) are passive lookups; leaving them in place is harmless.
